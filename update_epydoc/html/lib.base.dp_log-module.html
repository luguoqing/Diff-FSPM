Note:

  • debug: 调试信息
  • info: 程序执行关键节点的信息
  • success: 重要的成功信息
  • warning: 任务有错，可以恢复或忽略
  • diagnose: 出错诊断信息
  • error: 任务出错中断（程序含多个任务）
  • critical: 整个程序出错中断
  • info以上的日志必须是规整的，可以用来进行信息提取和统计
  • 屏幕输出 info以上级别的日志
  • 日志文件输出 所有级别的日志
  • wf日志文件输出warning以上级别的日志
